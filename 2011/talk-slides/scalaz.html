What is Scalaz?

A library for pure functional programming in Scala.

Scalaz 6: Current stable branch.

Scalaz 7: Redesigned library organization.

How do other people contribute?

  1. Want to do something in Scala, purely functionally.
  2. Find that there is nothing in the standard lib.
  3. Look for it in Scalaz. Ususally it's there.
  4. If not, borrow/invent an appropriate abstraction.
  5. Contribute it to scalaz.

What contributions do we welcome?

  • More examples
  • More tests
  • Better documentation
  • Useful, frequently-used functions
  • Purely functional data structures
  • More type classes
  • Type-class instances
  • Scalaz bindings for your project (e.g. scalaz-akka)
  • Complete rewrites

Pimps

Identity (*)

MA (*->*)

MAB (*->*->*)

Identity (*)

Methods like |+|, some, left, ===

  • For all types A, every value of type A implicitly converts to Identity[A]

MA (*->*)

Methods like map, flatMap, sequence, traverse.

  • For all types A and M[_], every value of type M[A] implicitly converts to MA[M,A]

MAB (*->*->*)

Mainly has arrow combinators

  • For all types A, B, and M[_,_], every value of type M[A,B] implicitly converts to MAB[M,A,B]

Wrapper types

  • BooleanW
  • OptionW
  • Function1W
  • etc.

Newtypes

For types that have many different implementations of the same type-class. For example, Int has Monoid for multiplication and addition.

case class IntMultiplication(value: Int) extends Newtype[Int]

Implicitly unwrapped, but not implicitly wrapped.

Scalaz 7

Version 7 has a redesigned organization.

  • Identity becomes *
  • MA becomes *->*
  • Composition vs. inheritance
  • Monad transformers
  • More mature Iteratees
  • More mature effect tracking
